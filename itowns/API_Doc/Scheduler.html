Scheduler

new Scheduler()

Source:
  • Core/Scheduler/Scheduler.js, line 109
The Scheduler is in charge of managing the Providers that are used to gather resources needed to display the layers on a View. There is only one instance of a Scheduler per webview, and it is instanciated with the creation of the first view.

Methods

addProtocolProvider(protocol, provider)

Source:
  • Core/Scheduler/Scheduler.js, line 269
Adds a provider for a specified protocol. The provider will be used when executing the queue to provide resources. See Provider for more informations. By default, some protocols are already set in iTowns: WMTS, WMS, WFS, TMS, XYZ, PotreeConverter, Rasterizer, 3D-Tiles and Static.

Warning: if the specified protocol has already a provider attached to it, the current provider will be overwritten by the given provider.
Parameters:
Name Type Description
protocol string The name of the protocol to add. This is the protocol parameter put inside the configuration when adding a layer. The capitalization of the name is not taken into account here.
provider Provider The provider to link to the protocol, that must respect the Provider interface description.
Throws:
an error if any method of the Provider is not present in the provider.

Error
