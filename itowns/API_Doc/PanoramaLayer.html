PanoramaLayer

new PanoramaLayer(id, coordinates, type, configopt)

Source:
  • Core/Prefab/Panorama/PanoramaLayer.js, line 20
Deprecated:
  • This layer is going to be removed after 2.7.0, along with StaticSource and PanoramaView. See https://github.com/iTowns/itowns/issues/739 See https://github.com/iTowns/itowns/issues/901
A TiledGeometryLayer to use with a PanoramaView. It has specific method for updating and subdivising its grid.
Parameters:
Name Type Attributes Description
id string The id of the layer, that should be unique. It is not mandatory, but an error will be emitted if this layer is added a View that already has a layer going by that id.
coordinates Coordinates The coordinates of the origin of the panorama.
type number The type of projection for the panorama: 1 for a cylindrical projection, 2 for a spherical one.
config Object <optional>
 Optional configuration, all elements in it will be merged as is in the layer. For example, if the configuration contains three elements name, protocol, extent, these elements will be available using layer.name or something else depending on the property name.
Properties
Name Type Attributes Default Description
object3d THREE.Object3d <optional>
 THREE.Group The object3d used to contain the geometry of the TiledGeometryLayer. It is usually a THREE.Group, but it can be anything inheriting from a THREE.Object3d.
maxSubdivisionLevel number <optional>
 10 Maximum subdivision level for this tiled layer.
ratio number <optional>
 1 Ratio for building the panorama sphere.

defineLayerProperty(propertyName, defaultValue, onChangeopt)

Source:
  • Layer/Layer.js, line 119
Inherited From:
  • Layer#defineLayerProperty
Defines a property for this layer, with a default value and a callback executed when the property changes.

When changing the property, it also emits an event, named following this convention: ${propertyName}-property-changed, with ${propertyName} being replaced by the name of the property. For example, if the added property name is frozen, it will emit a frozen-property-changed.

Example

The emitted event has some properties assigned to it

event = {
    new: {
        ${propertyName}: * // the new value of the property
    },
    previous: {
        ${propertyName}: * // the previous value of the property
    },
    target: Layer // the layer it has been dispatched from
    type: string // the name of the emitted event
}
Parameters:
Name Type Attributes Description
propertyName string The name of the property, also used in the emitted event name.
defaultValue * The default set value.
onChange function <optional>
 The function executed when the property is changed. Parameters are the layer the property is defined on, and the name of the property.

preUpdate(context, sources) → {Array.<TileMesh>}

Source:
  • Layer/TiledGeometryLayer.js, line 126
Inherited From:
  • TiledGeometryLayer#preUpdate
Does pre-update work on the context:
  • update the colorLayers and elevationLayers
  • update the maxElevationLevel
Once this work is done, it returns a list of nodes to update. Depending on the origin of sources, it can return a few things:
  • if sources is empty, it returns the first node of the layer (stored as level0Nodes), which will trigger the update of the whole tree
  • if the update is triggered by a camera move, the whole tree is returned too
  • if source.layer is this layer, it means that source is a node; a common ancestor will be found if there are multiple sources, with the default common ancestor being the first source itself
  • else it returns the whole tree
Parameters:
Name Type Description
context Object The context of the update; see the MainLoop for more informations.
sources Set.<(GeometryLayer|TileMesh)> A list of sources to generate a list of nodes to update.
Returns:
The array of nodes to update.
Type
Array.<TileMesh>

subdivideNode(context, node)

Source:
  • Layer/TiledGeometryLayer.js, line 313
Inherited From:
  • TiledGeometryLayer#subdivideNode
Subdivides a node of this layer. If the node is currently in the process of subdivision, it will not do anything here. The subdivision of a node will occur in four part, to create a quadtree. The extent of the node will be divided in four parts: north-west, north-east, south-west and south-east. Once all four nodes are created, they will be added to the current node and the view of the context will be notified of this change.
Parameters:

context Object The context of the update; see the MainLoop for more informations.
