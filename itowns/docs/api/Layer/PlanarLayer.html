PlanarLayer

A TiledGeometryLayer to use with a PlanarView. It has specific method for updating and subdivising its grid.

Constructor

PlanarLayer ( id: string, extent: Extent, object3d: THREE.Object3d, config: Object, config.maxSubdivisionLevel: number, config.maxDeltaElevationLevel: number )

Parameters

  • id — The id of the layer, that should be unique. It is not mandatory, but an error will be emitted if this layer is added a View that already has a layer going by that id.
  • extent — The extent to define the layer within.
  • object3d (default: THREE.Group) — The object3d used to contain the geometry of the TiledGeometryLayer. It is usually a THREE.Group, but it can be anything inheriting from a THREE.Object3d.
  • config (optional) — Optional configuration, all elements in it will be merged as is in the layer. For example, if the configuration contains three elements name, protocol, extent, these elements will be available using layer.name or something else depending on the property name.
    • config.maxSubdivisionLevel (default: 5) — Maximum subdivision level for this tiled layer.
    • config.maxDeltaElevationLevel (default: 4) — Maximum delta between two elevations tile.

Properties

isPlanarLayer : boolean

Used to checkout whether this layer is a PlanarLayer. Default is true. You should not change this, as it is used internally for optimisation.

Methods

attach ( layer: Layer )

Attach another layer to this one. Layers attached to a GeometryLayer will be available in geometryLayer.attachedLayers.

Parameters

  • layer — The layer to attach, that must have an update method.

changeOpacity ( object: Object )

Change the opacity of an object, according to the value of the opacity property of this layer.

Parameters

  • object — The object to change the opacity from. It is usually a THREE.Object3d or an implementation of it.

defineLayerProperty ( propertyName: string, defaultValue: *, onChange: function )

Defines a property for this layer, with a default value and a callback executed when the property changes.

When changing the property, it also emits an event, named following this convention: ${propertyName}-property-changed, with ${propertyName} being replaced by the name of the property. For example, if the added property name is frozen, it will emit a frozen-property-changed.

Parameters

  • propertyName — The name of the property, also used in the emitted event name.
  • defaultValue — The default set value.
  • onChange (optional) — The function executed when the property is changed. Parameters are the layer the property is defined on, and the name of the property.

Example

The emitted event has some properties assigned to it
event = {
    new: {
        ${propertyName}: * // the new value of the property
    },
    previous: {
        ${propertyName}: * // the previous value of the property
    },
    target: Layer // the layer it has been dispatched from
    type: string // the name of the emitted event
}

delete ( )

All layer's meshs are removed from scene and disposed from video device.

detach ( layer: Layer ) : boolean

Detach a layer attached to this one. See attach to learn how to attach a layer.

Returns

Confirmation of the detachment of the layer.

Parameters

  • layer — The layer to detach.

pickObjectsAt ( view: View, coordinates: Object, radius: number ) : Array

Picking method for this layer. It uses the Picking#pickTilesAt method.

Returns

An array containing all targets picked under the specified coordinates.

Parameters

  • view — The view instance.
  • coordinates — The coordinates to pick in the view. It should have at least x and y properties.
  • radius — Radius of the picking circle.

preUpdate ( context: Object, sources: Set.<(GeometryLayer|TileMesh)> ) : Array.<TileMesh>

Does pre-update work on the context:
  • update the colorLayers and elevationLayers
  • update the maxElevationLevel
Once this work is done, it returns a list of nodes to update. Depending on the origin of sources, it can return a few things:
  • if sources is empty, it returns the first node of the layer (stored as level0Nodes), which will trigger the update of the whole tree
  • if the update is triggered by a camera move, the whole tree is returned too
  • if source.layer is this layer, it means that source is a node; a common ancestor will be found if there are multiple sources, with the default common ancestor being the first source itself
  • else it returns the whole tree

Returns

The array of nodes to update.

Parameters

  • context — The context of the update; see the MainLoop for more informations.
  • sources — A list of sources to generate a list of nodes to update.

subdivideNode ( context: Object, node: TileMesh )

Subdivides a node of this layer. If the node is currently in the process of subdivision, it will not do anything here. The subdivision of a node will occur in four part, to create a quadtree. The extent of the node will be divided in four parts: north-west, north-east, south-west and south-east. Once all four nodes are created, they will be added to the current node and the view of the context will be notified of this change.

Parameters

  • context — The context of the update; see the MainLoop for more informations.
  • node — The node to subdivide.

subdivision ( context: Object, layer: PlanarLayer, node: TileMesh ) : boolean

Test the subdvision of a node, compared to this layer.

Returns

True if the node is subdivisable, otherwise false.

Parameters

  • context — The context of the update; see the MainLoop for more informations.
  • layer — This layer, parameter to be removed.
  • node — The node to test.

update ( context: Object, layer: Layer, node: TileMesh ) : Object

Update a node of this layer. The node will not be updated if:
  • it does not have a parent, then it is removed
  • its parent is being subdivided
  • is not visible in the camera

Parameters

  • context — The context of the update; see the MainLoop for more informations.
  • layer — Parameter to be removed once all update methods have been aligned.
  • node — The node to update.
