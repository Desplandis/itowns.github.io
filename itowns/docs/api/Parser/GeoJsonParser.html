GeoJsonParser

The GeoJsonParser module provide a parse method that takes a GeoJSON in and gives an object formatted for iTowns containing all necessary informations to display this GeoJSON.

Static Methods

parse ( json: string, options: GeoJsonParser~GeoJsonParserOptions ) : Promise

  • json — The GeoJSON file content to parse.
  • options — Options controlling the parsing.
Parse a GeoJSON file content and return a FeatureCollection.

Types

Feature : Object

  • type: string — Geometry type, can be point, multipoint, linestring, multilinestring, polygon or multipolygon.
  • vertices: Array.<number> — All the vertices of the geometry.
  • normals: Array.<number> — All the normals of the geometry.
  • size: number — the number of values of the array that should be associated with a coordinates. The size is 3 with altitude and 2 without altitude.
  • crs: string — Geographic or Geocentric coordinates system.
  • geometry: Array.<FeatureGeometry> — The feature's geometry, as an array of FeatureGeometry.
  • extent: Extent — The 2D extent containing all the geometries composing the feature.
Similar to a feature in a GeoJSON, but adapted to iTowns.

FeatureCollection : Object

  • features: Array.<Feature> — The array of features composing the collection.
  • extent: Extent — The 2D extent containing all the features composing the collection.
  • crs: string — Geographic or Geocentric coordinates system.
  • isFeature: boolean — Used to check whether this is FeatureCollection.
An object regrouping a list of features and the extent of this collection.

FeatureGeometry : Object

  • extent: Extent — The 2D extent containing all the points composing the geometry.
  • indices: Array.<Object> — If this geometry is a linestring or a polygon, contains the indices that define the geometry. Objects stored in this array have two properties, an offset and a count. The offset is related to the overall number of vertices in the Feature.
  • properties: Object — Properties of the geometry. It can be anything specified in the GeoJSON under the properties property.
Similar to the geometry of a feature in a GeoJSON, but adapted to iTowns. The difference is that coordinates are stored in unique Array of number

GeoJsonParserOptions : Object

  • crsOut: string — The CRS to convert the input coordinates to.
  • crsIn: string — Override the data CRS.
  • filteringExtent: Extent (optional) — Optional filter to reject features outside of this extent.
  • buildExtent: boolean (optional) — If true the geometry will have an extent property containing the area covered by the geom
  • filter: function (optional) — Filter function to remove features
  • mergeFeatures: boolean (default: true) — If true all geometries are merged by type and multi-type
  • withNormal: boolean (default: true) — If true each coordinate normal is computed
  • withAltitude: boolean (default: true) — If true each coordinate altitude is kept
  • overrideAltitudeInToZero: boolean (optional) — If true, the altitude of the source data isn't taken into account for 3D geometry convertions. the altitude will be override to 0. This can be useful if you don't have a DEM or provide a new one when converting (with Layer.convert).
